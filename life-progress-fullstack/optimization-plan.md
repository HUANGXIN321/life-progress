# 人生进度 — TRAE AI 创造力大赛初赛优化计划

> 基于评分标准四维度（创新性 30% / 实用性 30% / 完成度 20% / 美观度 20%）的针对性优化方案

---

## 一、当前作品现状速览

| 维度 | 当前状态 | 预估得分 | 优化空间 |
|------|---------|---------|---------|
| 创新性 | 生命年轮可视化 + 纯前端零依赖架构 | 中上 | 需强化差异化叙事 |
| 实用性 | 5 大功能模块已跑通，API 已部署 | 中上 | 需补齐后端覆盖 + 数据持久化 |
| 完成度 | 核心链路可用，但边缘场景有 bug | 中等 | 需系统性的体验打磨 |
| 美观度 | 暖色调 UI + 微动效，但响应式有瑕疵 | 中上 | 需移动端适配 + 性能优化 |

---

## 二、四维度优化路线图

### 维度一：创新性（30%）— 目标：强化"不同于现有产品"的差异化

评分子项：需求创新 / 解决思路 / 技术创新

| 优先级 | 优化项 | 具体动作 | 预期效果 |
|--------|--------|---------|---------|
| P0 | **生命年轮算法可视化升级** | 将静态环形图改为动态 SVG 路径动画，根据用户输入的生日实时计算并绘制人生进度，支持拖拽调整预期寿命 | 从"装饰性图表"升级为"可交互的人生计算器"，直观传递产品核心价值 |
| P0 | **AI 辅助日记功能** | 在日记模块接入大模型 API（或 TRAE 内置能力），提供"心情分析""年度回顾生成""人生建议"三个 AI 功能入口 | 这是现有日记/记录类 App 极少具备的功能，形成显著差异化 |
| P1 | **数据故事化导出** | 将用户数据（足迹 + 日记 + 目标）自动生成为一份精美的"人生年度报告"HTML 页面，支持分享 | 从"数据存储"升级为"人生叙事"，解决思路上的突破 |
| P1 | **TRAE 能力深度运用展示** | 在"关于作品"页面增加"本作品由 TRAE 辅助开发"的专门章节，列出 TRAE 在代码生成、调试、部署中的具体贡献点 | 回应"技术创新"评分项中对 TRAE 能力运用的考察 |

---

### 维度二：实用性（30%）— 目标：让 Demo 从"能看"变成"想用"

评分子项：场景洞察 / 解决效果 / 落地可行性

| 优先级 | 优化项 | 具体动作 | 预期效果 |
|--------|--------|---------|---------|
| P0 | **补齐后端 API 覆盖** | 为 `diaries`（日记）、`timeline`（足迹）、`goals`（目标）三个模块各增加 GET/POST API，与倾诉/分享一样支持跨设备互通 | 当前只有倾诉和分享有后端，个人数据完全本地。补齐后用户换设备也能看到全部数据，大幅提升实用价值 |
| P0 | **用户注册与持久化账户** | 将当前基于 localStorage 的"假登录"升级为真实用户系统（用户名 + 密码哈希存储），API 端对接用户认证 | 解决"当前 Demo 只是前端演示"的问题，让评委看到这是一个可真实使用的产品 |
| P1 | **数据备份与恢复** | 增加"导出 JSON 备份"和"从 JSON 导入恢复"功能，在设置页面提供一键操作 | 解决 IndexedDB 数据易丢失的真实痛点，提升落地可行性 |
| P1 | **目标进度提醒机制** | 为目标模块增加"截止日期提醒"功能，在首页以温和的方式（非弹窗）展示即将到期的目标 | 从"记录"升级为"推动行动"，提升解决效果 |
| P2 | **使用引导 Onboarding** | 新用户首次进入时，增加 3 步引导流程（介绍生命年轮 -> 演示添加足迹 -> 鼓励写下第一篇日记） | 降低学习成本，让评委/用户第一时间感知产品价值 |

---

### 维度三：完成度（20%）— 目标：消灭 bug，打磨流畅体验

评分子项：核心链路 / 用户体验 / 价值传递

| 优先级 | 优化项 | 具体动作 | 预期效果 |
|--------|--------|---------|---------|
| P0 | **修复已知体验问题** | 系统排查并修复以下问题：① 移动端布局错位 ② 模态框打开时背景仍可滚动 ③ 表单提交无 loading 状态 ④ 空状态图标在某些主题下对比度不足 | 消除"明显 bug 或卡顿"的扣分点 |
| P0 | **核心链路端到端测试** | 手动走通以下完整链路并记录：注册 -> 登录 -> 添加足迹 -> 写日记 -> 设定目标 -> 发倾诉 -> 回应倾诉 -> 分享故事 -> 点赞 -> 导出数据 -> 退出登录 | 确保评委试用时每个功能都能正常工作 |
| P1 | **加载性能优化** | ① 将 Google Fonts 改为 font-display: swap ② 图片添加 lazy loading ③ CSS 中减少重绘属性（box-shadow 动画改用 transform） | 首屏加载 < 2s，操作反馈 < 200ms |
| P1 | **错误边界与降级处理** | 为 API 请求增加统一的错误处理：网络失败时显示友好提示（而非空白或报错），并提供"重试"按钮 | 提升鲁棒性，避免评委遇到网络波动时体验崩塌 |
| P2 | **键盘导航全链路覆盖** | 确保所有交互元素（包括自定义下拉菜单、日期选择器、富文本区域）都可通过键盘操作 | 强化 ARIA 可访问性的完整度 |

---

### 维度四：美观度/设计体验（20%）— 目标：专业级视觉呈现

评分子项：视觉设计 / 交互友好度

| 优先级 | 优化项 | 具体动作 | 预期效果 |
|--------|--------|---------|---------|
| P0 | **移动端响应式全面适配** | 针对 375px - 768px 宽度区间，重新调整：① 生命年轮环形图尺寸 ② 卡片网格从 2 列改为 1 列 ③ 底部固定导航栏替代桌面端侧边按钮 | 当前移动端体验有明显瑕疵，这是扣分重灾区 |
| P0 | **交互动效一致性打磨** | 统一全站的动效语言：① 所有卡片 hover 效果统一为"微上浮 + 阴影扩散" ② 所有按钮统一为"涟漪反馈" ③ 页面切换统一为"淡入 + 微位移" | 让交互体验从"有动效"升级为"有设计体系的动效" |
| P1 | **空状态插画升级** | 为 5 个模块的空状态各生成一张风格统一的 AI 插画（当前只有图标），与暖色调主题保持一致 | 空状态是评委第一眼看到的画面，插画能大幅提升视觉专业度 |
| P1 | **主题切换过渡动画** | 主题切换时增加 0.3s 的 CSS 变量过渡动画，让颜色变化更柔和自然 | 细节处的精致感 |
| P2 | **加载骨架屏** | 为数据加载区域（足迹列表、日记列表等）添加骨架屏占位，替代当前的空白或 spinner | 减少等待焦虑，提升感知性能 |

## 五、风险与应对

| 风险 | 概率 | 应对方案 |
|------|------|---------|
| AI 日记功能调用 API 需要密钥，部署后可能暴露 | 中 | 使用 TRAE 环境变量或代理接口，不在前端暴露密钥 |
| 移动端适配改动量大，可能引入新 bug | 中 | 使用 Chrome DevTools 设备模拟器逐像素对比，每改一处验证一处 |
| 新增 API 导致 IGA Pages 构建再次失败 | 低 | 继续使用短路径构建策略，手动创建 index.mjs 作为 fallback |
| 时间不足，无法完成全部 P0 项 | 中 | 严格按优先级执行，P0 必须完成，P1/P2 视时间取舍 |

## 六、立即可执行的第一项任务

**任务：修复移动端响应式布局**

这是当前最影响"美观度"和"完成度"评分的问题。建议立即执行：

1. 打开 Chrome DevTools，切换到 iPhone SE (375px) 和 iPad Mini (768px) 视图
2. 记录所有布局错位、文字截断、按钮重叠的问题
3. 在 `index.html` 的 `<style>` 中增加 `@media (max-width: 768px)` 媒体查询
4. 重点修复：生命年轮尺寸、卡片网格列数、底部导航、模态框宽度

预计耗时 3-4 小时，完成后移动端体验将有质的飞跃。
